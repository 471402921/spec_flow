# PRD-Lite 质量检查清单

## 检查流程

按以下顺序逐项检查，问题按严重级别分类：
- **ERROR**：必须修复，会导致开发无法正确执行
- **WARNING**：建议修复，可能导致歧义或遗漏
- **INFO**：改进建议，提升文档质量

---

## 1. 结构完整性检查（ERROR）

- [ ] 8 个章节是否齐全（概述、角色与权限、核心实体与关系、功能需求、业务规则、验收标准、交付约束与分期策略、未来扩展）
- [ ] 文档头是否包含：版本、状态、更新日期
- [ ] §1 是否明确了"文档范围"和"不覆盖"

## 2. 角色与权限检查（ERROR）

- [ ] 所有角色是否都有"获得方式"说明
- [ ] 角色间的关系是否清晰（如"角色 A 也拥有角色 B 的所有权限"）
- [ ] 权限矩阵是否按操作对象分表
- [ ] 权限矩阵中的操作是否覆盖了 §4 功能需求中的所有关键操作
- [ ] 是否区分了"角色"和"关系"（如"宠物主人"是关系而非角色）

## 3. 实体与关系检查（WARNING）

- [ ] 每个实体是否有一句话描述
- [ ] 实体关系图是否标注了 1-N / N-M
- [ ] 关系规则是否说明了归属、共享、可选性
- [ ] 是否只包含业务实体（排除技术实体）

## 4. 功能需求检查（ERROR）

- [ ] 是否按功能领域分组
- [ ] 是否使用 FR-4.x.x 编号格式
- [ ] 是否只描述业务行为（不含 API 路径、HTTP 方法、数据库字段名等技术细节）
- [ ] 必填/选填字段是否明确
- [ ] 格式约束是否具体（如"至少 8 位，包含字母和数字"而非"密码需复杂"）
- [ ] 默认值是否明确

## 5. 业务规则检查（ERROR）

### 5.1 关键约束

- [ ] 是否使用 BR-xx 编号
- [ ] 每条规则是否只包含一个约束（避免复合规则）
- [ ] 数值是否具体（如"上限 20"而非"有上限"）
- [ ] 枚举值是否列出所有选项
- [ ] 是否同时包含"优先级"和"批次"两列
- [ ] 优先级和批次的差异是否合理（如高优先级规则延后到 P1 是否有依赖原因）

### 5.2 边界场景

- [ ] 是否覆盖了常见异常情况
- [ ] 处理方式是否明确（拒绝/允许/提示内容）
- [ ] 提示内容是否具体（如"提示'邀请码已过期，请联系家庭主人重新生成'"而非"提示错误"）

## 6. 规则与验收交叉检查（ERROR）

- [ ] **规则覆盖**：每条 BR 是否至少被一条 AC 覆盖
- [ ] **验收可追溯**：每条 AC 的"覆盖规则"列是否填写正确
- [ ] **批次一致**：BR 的批次与 AC 的分组是否一致（P0 规则的 AC 应在 P0 验收组）
- [ ] **无孤儿 AC**：是否存在覆盖规则为空且无法关联到任何 BR 的 AC（可以填 `-` 但应该有合理的业务场景支撑）

## 7. 验收标准检查（WARNING）

- [ ] 是否按批次分组（P0/P1/P2）
- [ ] 是否使用 AC-xx 编号
- [ ] 验收描述是否使用"用户可以..."或"用户不能..."句式
- [ ] 验收方式是否标注（自动/手工）
- [ ] 每条 AC 是否可独立测试（不依赖其他 AC 的结果）

## 8. 交付约束检查（ERROR）

### 7.1 分期策略

- [ ] 分期表是否包含四列：批次、范围、功能需求、说明
- [ ] 功能需求列是否引用了 FR 编号
- [ ] 批次间的依赖关系是否在注意事项中说明

### 7.2 交付约束

- [ ] 是否按批次列出（每个批次都有）
- [ ] 每批次是否包含三项：必须映射的规则、必须覆盖的验收项、必须覆盖的边界场景
- [ ] 规则编号是否与 §5 一致
- [ ] 验收项范围是否与 §6 一致
- [ ] 边界场景是否与 §5.2 中的场景对应

## 9. 内容一致性检查（WARNING）

- [ ] §4 中提到的规则是否都在 §5 中有对应的 BR 条目
- [ ] §2 权限矩阵中的角色是否与 §2.1 角色定义一致
- [ ] §3 实体关系是否与 §4 功能需求中描述的行为一致
- [ ] §7.1 分期策略中的 FR 编号是否覆盖了 §4 中的所有 FR

## 10. 写作质量检查（INFO）

- [ ] 是否避免了技术术语（如 API、数据库、索引、事务等）
- [ ] 数值约束是否都有单位（如"7 天"、"60 秒"、"5MB"）
- [ ] 时间相关规则是否明确了时区（如"以 UTC 时区为准"）
- [ ] 规则数量是否在 10~20 条之间（过多建议精简，过少建议补充）
- [ ] 边界场景是否充分（建议 10+ 条）

## 审核报告输出格式

```markdown
## PRD 质量审核报告

### 审核文件
- 文件：<文件路径>
- 版本：<PRD 版本>

### 发现问题

#### [ERROR] 必须修复
1. **[PRD-E-001]** §X <章节名>
   - 问题：描述
   - 建议：修复方案

#### [WARNING] 建议修复
1. **[PRD-W-001]** §X <章节名>
   - 问题：描述
   - 建议：修复方案

#### [INFO] 改进建议
1. **[PRD-I-001]** §X <章节名>
   - 问题：描述
   - 建议：修复方案

### 覆盖率统计

| 指标 | 数值 |
|------|------|
| BR 总数 | X |
| AC 总数 | X |
| BR 被 AC 覆盖率 | X% |
| 边界场景数 | X |

### 总结
- 问题统计：X 个 ERROR, Y 个 WARNING, Z 个 INFO
- 整体评价
- 优先修复建议
```
