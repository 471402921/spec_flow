# Phase 3 应用层功能部署实录

> 记录日期：2026-02-07
> 部署目标：完成 SOP Phase 3（最小可运行应用），新增 OpenAPI/Swagger、日志系统、TraceId 追踪、环境分离配置等功能

---

## 1. 概述

本次部署在已有的 Docker Compose 环境基础上（Phase 2），为 Spring Boot 应用添加应用层基础功能，使其成为一个具备完整可观测性和文档化的生产就绪应用。

**核心目标**：
- ✅ API 文档自动生成（Swagger UI）
- ✅ 结构化日志输出（Logback + 环境分离）
- ✅ 请求追踪（TraceId 自动注入）
- ✅ 监控端点（Spring Actuator）
- ✅ 自动化部署脚本

## 2. 新增功能清单

### 2.1 SpringDoc OpenAPI (v2.7.0)

**功能**：自动生成 REST API 文档

**访问地址**：
- Swagger UI：`/swagger-ui/index.html`
- OpenAPI Docs：`/api-docs`

**配置文件**：`OpenApiConfig.java`
- 配置 API 标题、版本、描述

**验证**：
```bash

# 公网
curl https://api.soulpal.me/swagger-ui/index.html
```

---

### 2.2 日志系统 (Logback)

**功能**：结构化日志输出，支持环境分离

**特性**：
- 彩色日志输出（开发环境）
- TraceId 自动注入 MDC
- 环境分离配置（dev/prod 不同日志级别）
- 日志格式：`时间戳 + 级别 + PID + 线程 + [traceId] + Logger + 消息`

**配置文件**：
- `logback-spring.xml`：主配置文件
- `application-dev.yml`：开发环境（DEBUG 日志 + ANSI 彩色）
- `application-prod.yml`：生产环境（WARN 日志 + 无彩色）

**日志示例**：
```
2026-02-07 10:15:30.123  INFO 12345 [nio-8080-exec-1] [a1b2c3d4e5f6] c.s.a.controller.HealthController : Hello endpoint called
```

---

### 2.3 TraceId 追踪

**功能**：为每个 HTTP 请求生成唯一标识，便于日志追踪和问题排查

**实现方式**：
- `TraceIdInterceptor`：Spring HandlerInterceptor
- 自动生成 UUID（去除 `-` 符号）
- 注入 SLF4J MDC（所有日志自动包含 traceId）
- 响应头自动添加 `X-Trace-Id`

**验证**：
```bash
curl -I http://localhost:8080/api/v1/hello
# 响应头包含：X-Trace-Id: a1b2c3d4e5f6...
```

---

### 2.4 环境配置分离

**配置结构**：
```
application.yml          # 基础配置（通用）
application-dev.yml      # 开发环境
application-prod.yml     # 生产环境
```

**环境切换**：
```bash
# 通过环境变量切换
SPRING_PROFILES_ACTIVE=prod

# Docker Compose 中配置
environment:
  SPRING_PROFILES_ACTIVE: ${SPRING_PROFILES_ACTIVE:-dev}
```

**配置差异**：

| 配置项 | dev | prod |
|--------|-----|------|
| 日志级别 | DEBUG | WARN |
| ANSI 彩色 | 开启 | 关闭 |
| 健康检查详情 | 始终显示 | 授权时显示 |

## 4. 自动化部署脚本

### 4.1 `deploy/deploy.sh`

**功能**：一键部署到 home-node

**执行流程**：
1. Git pull 最新代码
2. Docker compose down 停止旧服务
3. Docker compose up -d --build 构建并启动
4. 等待 15 秒服务启动
5. 执行健康检查验证
6. 输出服务状态和访问地址

**使用方式**：
```bash
# Mac 本地执行（远程部署）
ssh home-node "cd /srv/soulpal-service/deploy && ./deploy.sh"

# 或登录 home-node 后执行
cd /srv/soulpal-service/deploy
./deploy.sh
```

**成功输出示例**：

## 8. 常见问题

### 8.1 Swagger UI 访问 404

**问题**：访问 `/swagger-ui.html` 返回 404

**原因**：SpringDoc OpenAPI 的正确路径是 `/swagger-ui/index.html`（带 `/index.html`）

**解决**：使用正确路径 `/swagger-ui/index.html`

---

### 8.2 日志中没有 TraceId

**问题**：日志输出中看不到 `[traceId]`

**原因**：
1. 拦截器未注册
2. logback-spring.xml 中未配置 `%X{traceId}`

**解决**：
- 确认 `WebConfig` 中注册了 `TraceIdInterceptor`
- 确认 `logback-spring.xml` 中包含 `[%X{traceId}]`

---

### 8.3 环境变量不生效

**问题**：设置 `SPRING_PROFILES_ACTIVE=prod` 但仍然使用 dev 配置

**原因**：
1. 环境变量未传递到容器
2. 缓存问题

**解决**：
```bash
# 1. 检查 docker-compose.yml
environment:
  SPRING_PROFILES_ACTIVE: ${SPRING_PROFILES_ACTIVE:-dev}

# 2. 重新构建
docker compose down
docker compose up -d --build

# 3. 验证
docker compose exec api env | grep SPRING_PROFILES_ACTIVE
```
