## 3. 部署步骤

### 3.1 创建配置目录和文件

```bash
# SSH 连接到 home-node
ssh home-node

# 创建配置目录
sudo mkdir -p /etc/cloudflared

# 复制凭证文件
sudo cp ~/.cloudflared/b123f8b2-f3ac-4cfc-9b30-a07a0aec753d.json /etc/cloudflared/

# 设置文件权限（重要：需要 644，否则 cloudflared 无法读取）
sudo chmod 644 /etc/cloudflared/b123f8b2-f3ac-4cfc-9b30-a07a0aec753d.json
sudo chmod 644 /etc/cloudflared/config.yml
```

### 3.2 启动 cloudflared

```bash
# 使用 systemd 管理（已配置为系统服务）
sudo systemctl start cloudflared
sudo systemctl enable cloudflared   # 开机自启

# 查看服务状态
sudo systemctl status cloudflared
```

> **更新**: WSL2 重启 systemd 生效后，已从 `nohup` 切换到 systemd 服务管理。

### 3.3 验证启动状态

```bash
# 检查进程
ps aux | grep cloudflared

# 查看日志
cat /tmp/cloudflared.log

# 预期输出：
# INF Starting tunnel tunnelID=b123f8b2-f3ac-4cfc-9b30-a07a0aec753d
# INF Registered tunnel connection connIndex=0 ...
```

## 5. 常见问题

### 5.1 文件权限问题
**现象**：cloudflared 启动报错 `permission denied`
**解决**：将凭证文件权限设置为 644
```bash
sudo chmod 644 /etc/cloudflared/b123f8b2-f3ac-4cfc-9b30-a07a0aec753d.json
```

### 5.2 systemd 未生效
**现象**：`systemctl` 命令报错
**原因**：WSL2 配置 `systemd=true` 后需要重启 WSL2
**解决**：
```powershell
# 在 Windows PowerShell 中执行
wsl --shutdown
# 重新打开 WSL2
```

### 5.3 DNS 未生效
**现象**：域名无法解析
**解决**：
1. 检查 Cloudflare Dashboard 中 DNS 记录是否正确
2. 等待 DNS 传播（通常几分钟到几小时）
3. 本地刷新 DNS 缓存：`sudo dscacheutil -flushcache` (Mac)
