## 2. 创建的文件

### 2.1 `deploy/Dockerfile`

Multi-stage 构建：
- Stage 1 (builder): `eclipse-temurin:21-jdk` + Maven Wrapper 编译
- Stage 2 (runtime): `eclipse-temurin:21-jre` 仅运行 JAR

关键设计：
- 先 COPY pom.xml → `mvnw dependency:go-offline`（利用 Docker 层缓存加速后续构建）
- 再 COPY 整个模块目录（而非 `src` 子目录，避免空目录问题）
- 通过 `ARG MODULE=soulpal-api` 支持未来 worker 复用

### 2.2 `deploy/docker-compose.yml`

| 服务 | 镜像 | 端口 | 健康检查 |
|------|------|------|----------|
| postgres | `postgres:16` | 5432:5432 | `pg_isready -U soulpal` |
| redis | `redis:7-alpine` | 6379:6379 | `redis-cli ping` |
| api | 本地构建 | 8080:8080 | depends_on postgres/redis healthy |

- 网络：`soulpal-net` (bridge)
- 数据卷：`pgdata`、`redisdata`（命名卷，`docker compose down` 不删数据，`down -v` 才清）
- 环境变量：`${VAR:-default}` 模式，无 `.env` 也能跑

### 2.3 `deploy/.env.example`

```
POSTGRES_DB=soulpal
POSTGRES_USER=soulpal
POSTGRES_PASSWORD=soulpal_dev_2026
SPRING_PROFILES_ACTIVE=dev
```

### 2.4 `.dockerignore`（项目根目录）

排除 `.git`、`.idea`、`**/target`、`project_plan` 等不需要进入 Docker build context 的内容。

## 3. 前置条件配置

### 3.1 Docker 镜像加速器（中国网络必需）

```bash
# /etc/docker/daemon.json
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me"
  ]
}
```

配置后重启 Docker：
```bash
sudo systemctl restart docker
```

### 3.2 GitHub SSH 认证

```bash
# 在 home-node 上生成密钥
ssh-keygen -t ed25519 -C "home-node-deploy" -f ~/.ssh/home-node-deploy -N ""

# 配置 SSH 使用该密钥访问 GitHub
cat >> ~/.ssh/config <<EOF
Host github.com
  IdentityFile ~/.ssh/home-node-deploy
EOF

# 将公钥添加到 GitHub 账户
cat ~/.ssh/home-node-deploy.pub
```

### 3.3 Git 空目录处理

Git 不跟踪空目录，Maven 骨架中的空包目录需要添加 `.gitkeep`：
```
soulpal-common/src/main/java/com/soulpal/common/.gitkeep
soulpal-common/src/test/java/com/soulpal/common/.gitkeep
soulpal-api/src/test/java/com/soulpal/api/.gitkeep
soulpal-worker/src/test/java/com/soulpal/worker/.gitkeep
```

## 4. 部署步骤

### 4.1 首次部署


## 6. 遇到的问题与解决

### 6.1 Docker Hub 国内超时
- **现象**: `net/http: request canceled while waiting for connection`
- **解决**: 配置镜像加速器（见 3.1）

### 6.2 Docker 构建失败（空目录）
- **现象**: `failed to calculate checksum of ref: "/soulpal-common/src": not found`
- **原因**: Git 不跟踪空目录，`soulpal-common/src` 未被提交
- **解决**: 添加 `.gitkeep` + 改进 Dockerfile 为 `COPY soulpal-common soulpal-common`（整个模块目录）

### 6.3 SSH sudo 无 TTY
- **现象**: `sudo: a terminal is required to read the password`
- **解决**: 使用 `echo 'password' | sudo -S command` 模式

### 6.4 Docker daemon 未运行
- **现象**: `Cannot connect to the Docker daemon`
- **解决**: WSL2 需手动启动 `sudo service docker start`

## 7. 常用运维命令

```bash
# 查看容器状态
ssh home-node "cd /srv/soulpal-service/deploy && docker compose ps"

# 查看 API 日志
ssh home-node "cd /srv/soulpal-service/deploy && docker compose logs -f api"

# 重启单个服务
ssh home-node "cd /srv/soulpal-service/deploy && docker compose restart api"

# 停止全部服务
ssh home-node "cd /srv/soulpal-service/deploy && docker compose down"

# 停止并清除数据卷（慎用）
ssh home-node "cd /srv/soulpal-service/deploy && docker compose down -v"

# 备份数据库
ssh home-node "cd /srv/soulpal-service/deploy && docker compose exec postgres pg_dump -U soulpal soulpal > backup.sql"
```
