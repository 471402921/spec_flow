# SoulPal 用户模块 - 产品需求文档（PRD）

> 版本：v1.3 | 状态：定稿 | 更新日期：2026-02-12

---

## 1. 概述

本文档定义 SoulPal 用户模块的功能需求与业务规则，供开发团队完成数据库设计、接口设计和产品开发。

**文档范围**：用户注册登录、用户资料、宠物管理、家庭共享。

**不覆盖**：IoT 设备绑定、GPS/围栏/运动等业务功能、API 与数据模型设计。

---

## 2. 角色与权限

### 2.1 角色定义

系统中存在三种角色，一个用户可以同时拥有多种角色身份。例如：用户 A 既是自己宠物的主人（User），又是家庭 X 的创建者（Family Owner），同时还是家庭 Y 的成员（Family Member）。

| 角色 | 获得方式 | 说明 |
|------|---------|------|
| 用户（User） | 注册 | 所有注册用户，可独立管理自己的宠物 |
| 家庭主人（Family Owner） | 创建家庭 | 创建家庭的用户，拥有家庭管理权限 |
| 家庭成员（Family Member） | 被邀请加入家庭 | 可查看家庭内所有成员的宠物 |

> **说明**：
> - 家庭主人（Family Owner）也是该家庭的成员，拥有 Family Member 的所有权限，外加管理权限。
> - "宠物主人"不是独立角色，而是用户与宠物之间的归属关系描述 — 即创建该宠物的用户。

### 2.2 权限矩阵

**宠物相关权限**（以"操作者与某只宠物的关系"为维度）：

| 操作 | 宠物主人（创建者） | 同家庭成员 | 非家庭成员 |
|------|-------------------|-----------|-----------|
| 查看宠物信息 | ✅ | ✅ | ❌ |
| 编辑宠物信息 | ✅ | ❌ | ❌ |
| 删除宠物 | ✅ | ❌ | ❌ |

> **说明**：添加宠物是用户级操作（与现有宠物无关），所有注册用户均可为自己添加宠物，无需家庭。宠物始终归属于创建者。

**家庭相关权限**：

| 操作 | 家庭主人 | 家庭成员 |
|------|---------|---------|
| 生成邀请码 | ✅ | ❌ |
| 移除成员 | ✅ | ❌ |
| 转让主人身份 | ✅ | ❌ |
| 解散家庭 | ✅ | ❌ |
| 查看成员列表 | ✅ | ✅ |
| 查看家庭内宠物 | ✅ | ✅ |
| 退出家庭 | ❌（需先转让主人身份） | ✅ |

## 3. 核心实体与关系

### 3.1 实体

| 实体 | 说明 |
|------|------|
| 用户（User） | 注册用户，系统的基础身份 |
| 宠物（Pet） | 归属于用户，宠物的核心档案 |
| 家庭（Family） | 可选的共享单元，用于多人协作管理宠物 |

### 3.2 实体关系

```
User 1 ──── N Pet          一个用户可拥有多只宠物
User N ──── M Family       一个用户可加入多个家庭，一个家庭可有多个成员
```

- 宠物归属于用户（而非家庭），通过用户的家庭关系间接共享
- 家庭是可选的 — 用户不建家庭也可以独立管理自己的宠物
- 一个用户在同一个家庭中只能有一个角色（Owner 或 Member）
- **宠物共享规则**：用户的全部宠物对其所在的每个家庭的所有成员可见。不支持选择性共享（即不能选择哪些宠物在哪个家庭中可见）

## 4. 功能需求

### 4.1 用户注册与登录

#### FR-4.1.1 注册

- 使用邮箱 + 密码注册
- 注册时必填字段：邮箱、密码
- 注册时选填字段：昵称（不填则默认为邮箱 @ 前缀，如 `user@example.com` → `user`）
- 邮箱格式校验（RFC 5322），不允许重复注册
- 邮箱存储时统一转为小写（`User@Example.com` → `user@example.com`）
- 密码要求：至少 8 位，必须同时包含字母和数字
- 注册后发送验证邮件，用户点击邮件中的链接完成邮箱验证
- 验证链接有效期：24 小时，一次性使用，过期需重新发送
- 同一邮箱 60 秒内只能发送一次验证邮件（防刷）
- 未验证邮箱的用户可以登录，但以下功能受限：创建家庭、加入家庭

#### FR-4.1.2 登录

- 使用邮箱 + 密码登录
- 邮箱匹配不区分大小写
- 登录成功返回 Session Token
- Session Token 有效期：30 天。超时后需重新登录
- 连续 5 次密码错误，锁定账号 15 分钟（锁定期间即使密码正确也无法登录）
- 锁定计数在登录成功后重置为 0
- 不限制同时登录设备数

#### FR-4.1.3 密码重置

- 用户提供注册邮箱，系统发送重置链接
- 同一邮箱 60 秒内只能发送一次重置邮件（防刷）
- 重置链接有效期 30 分钟，一次性使用
- 重置成功后，所有已有 Session 失效（强制其他设备重新登录）
- 如果邮箱未注册，不返回错误信息（防止邮箱枚举攻击），静默处理

#### FR-4.1.4 登出

- 使当前 Session Token 失效
- 不影响其他设备的 Session

### 4.2 用户资料管理

#### FR-4.2.1 个人信息

- 可编辑字段：昵称、头像
- 昵称长度限制：2-20 个字符
- 头像支持上传图片（格式：JPG/PNG，大小上限：5MB）
- 头像存储方式：上传到对象存储（如 S3），数据库存储 URL 字符串

#### FR-4.2.2 账号安全

- **修改密码**：需验证当前密码，新密码规则同注册（至少 8 位，包含字母和数字）
- **修改邮箱**：
  1. 用户输入当前密码 + 新邮箱地址
  2. 系统验证当前密码正确
  3. 系统检查新邮箱是否已被占用
  4. 系统发送验证链接到新邮箱
  5. 用户点击验证链接后，邮箱更新生效，旧邮箱立即失效
  6. 验证链接有效期：24 小时，一次性使用
  7. 用户必须使用新邮箱登录

#### FR-4.2.3 注销账号

- 需验证当前密码
- 注销后：所有宠物数据标记为删除、退出所有家庭、所有 Session 失效
- 如果是家庭主人（任意家庭），必须先处理所有家庭主人身份（转让或解散），才能注销
- 注销为软删除，30 天内可联系客服申请恢复

### 4.3 宠物管理

#### FR-4.3.1 添加宠物

- 每个用户最多拥有 20 只宠物（BR-02），超出时拒绝并提示"已达宠物数量上限"
- 必填字段：
  - **种类**（species）：枚举值 `DOG`、`CAT`
  - **名字**（name）：1-30 个字符
  - **品种**（breed）：自由文本输入，1-50 个字符
  - **性别**（gender）：枚举值 `MALE`、`FEMALE`
- 选填字段：
  - **生日**（birthday）：日期类型，不能晚于当天（以 UTC 时区为准）
  - **头像**（avatar）：JPG/PNG，大小上限 5MB，存储方式同用户头像（对象存储 URL）
- 宠物自动归属于创建者
- 添加时系统检查该用户是否有软删除状态的同名同种类宠物，如有则提示可恢复（详见 FR-4.3.5）

#### FR-4.3.2 编辑宠物

- 仅宠物主人（创建者）可编辑
- 可修改所有字段（种类、名字、品种、性别、生日、头像）

#### FR-4.3.3 删除宠物

- 仅宠物主人可删除
- 删除需客户端二次确认（"确定要删除宠物 XX 吗？删除后 30 天内可恢复"）
- 删除后，家庭内其他成员也将无法看到该宠物
- 删除为软删除，记录删除时间戳（详见 BR-06）

#### FR-4.3.4 查看宠物列表

- 默认展示：自己创建的所有宠物（未加入任何家庭时，只有这部分）
- 如果加入了家庭：额外展示家庭内其他成员的宠物（需标注所属主人昵称）
- 如果加入了多个家庭：按家庭分组展示
- 同一只宠物如果在多个家庭中可见（因为宠物主人同时在多个家庭），每个家庭分组中各出现一次

#### FR-4.3.5 重新添加已删除的宠物

- 用户提交添加宠物请求时，系统检查该用户是否有软删除状态的宠物与新宠物的名字和种类匹配
- 如果匹配到一条：提示"发现已删除的同名宠物，是否恢复之前的数据？"
- 如果匹配到多条：列出所有匹配的已删除宠物（显示名字、种类、删除时间），让用户选择恢复哪一只，或选择创建新宠物
- 恢复操作：取消软删除标记，宠物信息恢复到删除前的状态
- 恢复后用户可通过编辑功能（FR-4.3.2）修改宠物信息
- 恢复操作不消耗新的宠物配额（恢复的宠物算在已有数量内）

### 4.4 家庭管理

#### FR-4.4.1 创建家庭

- 每个用户最多加入 5 个家庭（含自己创建的）（BR-03），超出时拒绝并提示"已达家庭数量上限"
- 用户可创建家庭，需提供家庭名称
- 家庭名称长度限制：2-20 个字符
- 家庭名称可重复（不同家庭允许同名）
- 创建者自动成为家庭主人（Family Owner）
- 需已完成邮箱验证

#### FR-4.4.2 生成邀请码

- 仅家庭主人可生成
- 邀请码为 8 位字符，字符集为大写字母 + 数字，排除易混淆字符（0/O/I/1），有效字符集共 32 个：`ABCDEFGHJKLMNPQRSTUVWXYZ23456789`
- 邀请码全局唯一（跨家庭不重复），生成后需数据库唯一性校验
- 有效期 7 天，过期后自动失效
- 同一时间只能存在一个有效邀请码，生成新码时旧码立即失效

#### FR-4.4.3 加入家庭

- 用户输入邀请码加入家庭
- 邀请码匹配不区分大小写（输入自动转大写）
- 加入后角色为家庭成员（Family Member）
- 需已完成邮箱验证
- 校验规则（按顺序检查）：
  1. 邀请码不存在 → 提示"邀请码无效"
  2. 邀请码已过期 → 提示"邀请码已过期，请联系家庭主人重新生成"
  3. 用户已是该家庭成员 → 提示"你已是该家庭成员"
  4. 用户已加入 5 个家庭 → 提示"已达加入家庭数量上限"
  5. 家庭已达成员上限（10 人） → 提示"家庭成员已满"

#### FR-4.4.4 查看家庭

- 查看家庭成员列表（显示昵称、头像、角色）
- 查看家庭内所有成员的宠物
- 家庭名称可由家庭主人修改（长度限制同创建：2-20 个字符）

#### FR-4.4.5 移除成员

- 仅家庭主人可操作
- 家庭主人不能移除自己
- 被移除的成员立即失去对家庭内宠物的查看权限
- 被移除成员的宠物归属不变（仍属于该成员）
- 被移除的成员可通过新的邀请码重新加入（无黑名单机制）

#### FR-4.4.6 转让主人身份

- 家庭主人可将主人身份转让给任一家庭成员
- 转让后，原主人变为普通成员

#### FR-4.4.7 退出家庭

- 家庭成员可主动退出
- 家庭主人不能退出（需先转让主人身份）
- 退出后，该用户的宠物跟随用户离开，家庭内其他成员不再可见

#### FR-4.4.8 解散家庭

- 仅家庭主人可操作
- 解散需客户端二次确认
- 解散后：所有成员关系解除，各用户的宠物归属不变
- 解散为硬删除（家庭及关联的成员关系、邀请码数据物理删除）

## 5. 业务规则

### 5.1 关键约束

| 编号 | 规则 | 优先级 | 批次 |
|------|------|--------|------|
| BR-01 | 邮箱全局唯一，不区分大小写，存储时统一转为小写 | P0 | P0 |
| BR-02 | 一个用户可拥有的宠物数量上限：20 | P0 | P0 |
| BR-03 | 一个用户可加入的家庭数量上限：5（包含自己创建的家庭） | P0 | P1 |
| BR-04 | 一个家庭的成员数量上限：10（包含家庭主人） | P0 | P1 |
| BR-05 | 邀请码有效期 7 天，同一家庭同时只存在一个有效邀请码，全局唯一 | P0 | P1 |
| BR-06 | 删除宠物为软删除（记录删除时间戳），数据保留 30 天后物理清除；30 天内用户重新添加同名同种类宠物时可选择恢复 | P0 | P0 |
| BR-07 | 注销账号为软删除（记录删除时间戳），数据保留 30 天后物理清除（期间可联系客服恢复） | P1 | P2 |
| BR-08 | 密码使用 bcrypt 加密存储，不可逆 | P0 | P0 |
| BR-09 | 宠物种类 v1 仅支持 `DOG`、`CAT` 两个枚举值 | P0 | P0 |
| BR-10 | 宠物性别为 `MALE`、`FEMALE` 两个枚举值 | P0 | P0 |
| BR-11 | 邮箱验证链接和邮箱修改验证链接有效期均为 24 小时，一次性使用 | P1 | P2 |
| BR-12 | 密码重置链接有效期 30 分钟，一次性使用 | P1 | P2 |
| BR-13 | 账号锁定：连续 5 次密码错误锁定 15 分钟，登录成功后重置计数 | P1 | P2 |
| BR-14 | Session Token 有效期 30 天 | P0 | P0 |
| BR-15 | 邮件发送频率限制：同一邮箱 60 秒内最多发送一次（验证邮件、重置邮件均适用） | P1 | P2 |
| BR-16 | 头像存储方式：对象存储（如 S3），数据库存 URL 字符串 | P0 | P0 |
| BR-17 | 解散家庭为硬删除（家庭、成员关系、邀请码物理删除） | P0 | P1 |
| BR-18 | 日期校验时区统一使用 UTC | P0 | P0 |

### 5.2 边界场景

| 场景 | 处理方式 |
|------|---------|
| 用户已在家庭 A 中，又加入家庭 B | 允许（上限 5 个家庭）。宠物列表按家庭分组展示 |
| 用户在家庭中是唯一成员且为主人，想退出 | 不允许退出，只能解散家庭 |
| 家庭主人注销账号 | 阻止。需先处理所有家庭主人身份（转让或解散） |
| 邀请码已被使用一次，其他人再使用 | 允许。邀请码可多次使用，直到过期或家庭满员 |
| 用户用同一邀请码重复加入 | 拒绝。提示"你已是该家庭成员" |
| 用户删除了自己唯一的宠物 | 允许。用户仍可查看家庭内其他成员的宠物 |
| 家庭中所有宠物都被删除 | 家庭仍然存在，只是没有可查看的宠物 |
| 家庭已满员（10 人），有人使用有效邀请码 | 拒绝加入，提示"家庭成员已满" |
| 被移除的成员再次使用邀请码加入 | 允许。无黑名单机制 |
| 用户已加入 5 个家庭，再加入新家庭 | 拒绝。提示"已达加入家庭数量上限" |
| 用户已有 20 只宠物，再添加新宠物 | 拒绝。提示"已达宠物数量上限" |
| 用户注册时使用大小写不同的已注册邮箱 | 拒绝。邮箱统一转小写后判重 |
| 密码重置邮箱未注册 | 不返回错误，静默处理（安全考虑） |
| 用户在锁定期间尝试登录 | 拒绝。提示"账号已锁定，请 X 分钟后再试" |
| 邀请码不存在 | 拒绝。提示"邀请码无效" |
| 邀请码已过期 | 拒绝。提示"邀请码已过期，请联系家庭主人重新生成" |
| 家庭主人用自己邀请码加入自己家庭 | 拒绝。命中"已是家庭成员"检查 |
| 修改邮箱后旧邮箱登录 | 拒绝。邮箱更新后旧邮箱立即失效 |
| 用户同时是多个家庭主人想注销 | 阻止。提示需处理所有家庭主人身份（列出相关家庭） |
| 用户 A 在家庭 X 和 Y，用户 B 只在 X | 用户 B 只能看到家庭 X 成员的宠物，无法看到家庭 Y 成员的宠物 |

## 8. 未来扩展（本版本不实现）

以下功能在架构设计中预留扩展点，但不在本版本实现：

- **朋友/关注者**：轻量级宠物分享，不需要加入家庭即可关注他人宠物
- **第三方登录**：微信、Google、Apple ID 登录
- **宠物健康档案**：体重记录、绝育状态、芯片号、疫苗记录
- **IoT 设备绑定**：智能项圈与宠物的关联管理
- **细粒度权限**：家庭成员可被授予编辑某只宠物的权限
- **更多宠物种类**：鸟类、爬行类等（当前仅支持 `DOG`、`CAT`）
- **选择性宠物共享**：允许用户选择在特定家庭中共享哪些宠物
