**里程碑**

**已完成日期**: 2026-02-07

**实际实现内容**:
1. **数据库配置**:
   - MyBatis-Plus 3.5.9（ASSIGN_UUID ID 生成策略、驼峰转换）
   - Flyway 自动迁移（baseline-on-migrate）
   - PostgreSQL 16 驱动配置
   - HikariCP 连接池配置

2. **specflow-common 模块**:
   - `BusinessException` - 业务异常基类（code + message + data）
   - `NotFoundException` - 资源未找到异常
   - `AuthenticationException` - 认证失败异常
   - `Result<T>` - 统一响应封装
   - `GlobalExceptionHandler` - 全局异常处理器

3. **Session 模块（DDD Light 4 层架构）**:
   - **Domain 层**（纯 POJO）:
     - `Session.java` - 领域实体（id、userId、token、expiredAt、revoked、createdAt、updatedAt）
     - `SessionRepository.java` - 仓储接口
     - 领域行为：`isExpired()`、`isValid()`、`revoke()`
     - 工厂方法：`create(userId, token, expirationDays)`
   - **Infrastructure 层**:
     - `SessionDO.java` - MyBatis-Plus 数据对象（@TableName、@TableId）
     - `SessionMapper.java` - MyBatis Mapper（extends BaseMapper）
     - `SessionConverter.java` - DO ↔ Entity 转换器（处理时区转换）
     - `SessionRepositoryImpl.java` - 仓储实现
   - **Application 层**:
     - `SessionService.java` - 应用服务（@Transactional、UseCase）
     - 支持创建、查询、验证、撤销会话
   - **Interfaces 层**:
     - `SessionController.java` - REST 控制器
     - `CreateSessionRequest.java`、`SessionResponse.java` - DTO

4. **数据库迁移脚本**:
   - `V1.0__init_schema.sql` - 创建 session 表（7 个字段、3 个索引、自动更新触发器）

- `specflow-api/src/main/resources/db/migration/V1.0__init_schema.sql`

**技术决策记录**:
- 选择简单 UUID Token 而非 JWT（降低初期复杂度，可后续演进）
- DO 使用 LocalDateTime，Entity 使用 Instant（统一使用 UTC 时区转换）
- MyBatis-Plus ASSIGN_UUID 策略（分布式友好）
- 逻辑删除（revoked 字段）而非物理删除（保留审计日志）

**遇到的问题与解决**:
1. Flyway 版本 10.22.1 不存在 → 移除显式版本，使用 Spring Boot 管理版本
2. 代码审核发现缺少 Lombok 注解 → 添加 @RequiredArgsConstructor、@Slf4j、@Data 等
3. 时区使用 systemDefault() 不规范 → 明确使用 ZoneId.of("UTC")
4. 生产环境配置有默认密码 → 移除 application-prod.yml 的默认值
5. **Docker 环境数据库连接失败** → application-dev.yml 硬编码 localhost，改为 `${DB_HOST:localhost}` 支持环境变量覆盖
6. **代码审查发现 4 个 WARNING 问题** → 修复 Boolean→boolean（NPE风险）、配置外部化、substring防护、@ApiResponse注解
7. **Lombok 生成器名称变化** → Boolean wrapper 生成 getRevoked()，boolean primitive 生成 isRevoked()
8. **单元测试 @Value 不生效** → 使用 ReflectionTestUtils.setField() 在测试中手动注入配置值
